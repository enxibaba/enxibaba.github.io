RunLoop 实践

RunLoop 的概念

通常一个线程一次只能执行一个任务，执行完成任务后线程就会退出。如果我们需要一个机制，让线程能随时处理事件但并不退出，通常的逻辑是这样的：

func loop() {
   init();
   do {
       var message = get_next_message();
       process_message(message);
   } while (message != quit)
 }

所以，RunLoop 实际上就是一个对象，这个对象管理了其需要处理的事件和消息，并提供了一个入口函数来执行上面的 Loop 函数的逻辑。线程执行了这个函数后，就一直处于这个函数内部 “接受消息->等待->处理” 的循环中，直到这个循环结束 （比如传入quit的消息），函数返回。
